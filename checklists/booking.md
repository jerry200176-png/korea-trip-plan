# Booking checklist

**Founder executes paid bookings.** Agent does not purchase or submit personal data.

## Before paying

- [ ] Dates locked (D1)
- [ ] Route locked (D2)
- [ ] Airports chosen (D3)
- [ ] Cancellation policy acceptable
- [ ] Double bed confirmed for lodging
- [ ] Location: transit access + evening convenience store nearby preferred

## After paying (offline only)

- [ ] Save confirmation to password manager / private note
- [ ] Update local `bookings.local.yaml` if used for PDF merge
- [ ] **Do not** git commit confirmation codes or QR tickets
- [ ] Add public-safe placeholders only to example files if structure changes
